# .NET MAUI 跨平台開發學習報告

## 課程概述

本專案以 .NET MAUI 為核心，透過 12 個循序漸進的模組，從畫面設計、資料管理到後端整合，完整體驗一款跨平台行動應用程式從無到有的開發流程。

---

## 學習歷程與收穫

### 一、體會「一次開發，多平台執行」的便利

過去開發行動應用，往往要為 Android 與 iOS 各寫一份程式。MAUI 採用「單一專案」架構，讓同一份程式碼能編譯到 Android、iOS、macOS 與 Windows，大幅降低維護成本，也讓注意力能集中在功能本身，不必反覆處理平台差異。

### 二、用 XAML 設計畫面與管理素材

透過版面配置與控制項的練習，學會用 XAML 這種「描述式語法」快速排版，並掌握以下實務技巧：

- **依平台調整外觀**：同一個畫面能依 Android、iOS 或 Windows 自動套用不同設定。
- **集中管理素材**：字型、圖片統一放在 `shared/Resources/` 共用資料夾，各模組以連結方式引用，避免重複又易於維護。

### 三、認識 MVVM：讓畫面與邏輯分家

進入 mod07、mod08 後，開始接觸 MVVM 模式。這個觀念把「畫面」與「邏輯」分成兩部分，讓程式更乾淨、更容易維護。搭配 CommunityToolkit.Mvvm 工具包，許多重複的程式碼可以自動產生，撰寫 ViewModel 變得相當輕鬆。

### 四、學會依賴注入：讓元件可以靈活替換

mod09 引入「依賴注入」的觀念，把畫面、邏輯、資料服務交給框架統一管理。最大的好處是：當資料來源從本機切換到遠端 API（mod11），上層程式幾乎不用改動，後續維護與測試都更有彈性。

### 五、後端整合：自己動手做 API

mod10 切換到後端，使用 ASP.NET Core 與 EF Core 建立一組寵物資料的 API，並透過 SQL Server 儲存資料。這部分讓我具備了從零打造後端服務的能力，也理解前後端如何分工。

### 六、串接前後端：讓手機真的連到伺服器

mod11 把前面所有學到的串成一個完整流程：MAUI 應用程式透過 `HttpClient` 呼叫 mod10 提供的 API，完成新增、查詢、修改、刪除等操作，並能正確處理網路斷線等狀況。

---

> **特別收穫：學會讓手機 App 連接本機 API 伺服器**
>
> 課前曾多次嘗試從 MAUI 連接本機 API 卻始終失敗，直到 mod11 才釐清關鍵：Android 模擬器需以 `10.0.2.2` 取代 `localhost`，並針對各平台處理開發環境的自簽憑證信任問題，兩個盲點一次解決，對後續整合開發受益良多。

## 核心技術總結

| 技術領域 | 應用模組 | 學習重點 |
|----------|----------|----------|
| MAUI 跨平台架構 | mod01–mod11 | 一份程式碼、多平台輸出 |
| XAML 畫面設計 | mod02–mod06 | 版面配置、跨平台條件、樣式複用 |
| 資料 Binding | mod07–mod11 | 畫面與資料自動同步 |
| MVVM 模式 | mod08–mod11 | 畫面與邏輯分離、提升可維護性 |
| 依賴注入 | mod09–mod11 | 元件解耦、靈活替換實作 |
| ASP.NET Core Web API | mod10 | 設計 RESTful 後端服務 |
| EF Core + SQL Server | mod10 | 資料庫存取與初始資料種入 |
| HttpClient | mod11 | 連接遠端 API、處理網路與憑證 |
